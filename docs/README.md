# Webframeplayer Documentation

Welcome to the `docs` folder. This is the central repository for all project documentation. The structure is strictly organized to ensure maintainability and clarity for both human developers and AI agents.

## Directory Structure

- **`/docs/business/`**: Product vision, user stories, feature specifications, roadmap, phase plans, and general business logic rules.
- **`/docs/frontend/`**: UI architecture, screens, components, design systems, styling tokens, state management, and navigation rules.
- **`/docs/backend/`**: API endpoints, server architecture, external integrations, services, authentication flows, and business logic implementation details.
- **`/docs/db/`**: Database schema definitions, tables, relationships, constraints, and security policies (like RLS).
- **`/docs/migrations/`**: Database migration files, historical SQL scripts, and schema update logs.
- **`/docs/map/`**: Contains `map.md`, the codebase sitemap listing every important file and folder in the project.
- **`/docs/test/`**: Testing strategies, E2E test guides, unit test guides, and manual testing templates.
- **`/docs/tasks/`**: Archival folder for completed development tasks (planning, walkthrough, and verification reports).

## Key Files
- **[`ARCHITECTURE.md`](./ARCHITECTURE.md)**: The core architectural decisions, tech stack, and high-level system design.
- **`features.json`**: Current status configuration of the system's features.

**Note to Agents:** Always check [`AGENTS.md`](../AGENTS.md) at the root directory before starting work.
